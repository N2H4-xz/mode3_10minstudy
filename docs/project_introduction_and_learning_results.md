## 2. 运行环境与基本用法

项目使用 Python 编写，依赖见 `requirements.txt`：

## 3. 主流程说明

项目主入口为 `main.py`，完整流程如下：

1. 读取输入文本。
2. 初始化 OpenAI 兼容客户端。
3. 调用 `analyzer.py` 分析说话风格，生成 `StyleProfile`。
4. 调用 `prompt_builder.py` 生成两版提示词：
   - 版本 A：LLM 整理生成版。
   - 版本 B：模板拼接版。
5. 调用 `verifier.py`，使用固定验证问题测试两版提示词的回答效果。
6. 由用户判断是否满意。
7. 如不满意，调用 `feedback.py` 选择需要修正的特征，并只对这些特征重新分析。
8. 达到用户满意或到达最大迭代次数后输出最终提示词。

### 3.1 风格学习机制

本项目中的“学习风格”不是对模型参数进行微调，也不会训练一个新的模型。它采用的是“样本文本分析 + 结构化风格画像 + 提示词生成”的方式：先让大语言模型从单一说话人的文本中抽取可描述、可引用、可审计的表达规律，再把这些规律转换成可复用的 system prompt。

整个风格学习过程可以拆成四个层次。

第一层是样本准备。`main.py` 只接受纯文本输入，并且要求输入尽量来自同一个说话人。对于莫言、余华这类访谈数据，项目不会直接把完整访谈或 SRT 输入给分析器，而是先通过人工校对和 `tools/correct_moyan_yuhua_transcripts.py` 将访谈内容按说话人拆分，生成 `transcripts/inputs/input_style_moyan_yuhua_first30_moyan.txt`、`transcripts/inputs/input_style_moyan_yuhua_first30_yuhua.txt` 等单人样本。这样可以避免主持人、嘉宾和现场插话混入同一个风格画像。

第二层是特征抽取。`analyzer.py` 会把完整样本文本包在 `<chat_log>...</chat_log>` 中发送给模型，并使用专门的系统提示词要求模型分析整段文本，而不是只看开头几句。分析维度由 `config.py` 中的 `CHARACTERISTICS_SPEC` 固定为六类一级指标：

| 分类 | 项目 key | 具体指标 |
| --- | --- | --- |
| 词汇复杂度 | `lexical_complexity` | 平均词长、type-token ratio、高级词汇使用情况 |
| 句法复杂度 | `syntactic_complexity` | 平均句子长度、从句出现频率、词性分布 |
| 形式化指数 | `formality_indices` | 代词比例、功能词比例、正式表达与口语表达占比 |
| 情感表达度 | `emotiveness` | 感叹号、表情符号、强化词或情感词汇的使用频率 |
| 可读性指标 | `readability_metrics` | 长句比例、分句密度、抽象词密度、结构清晰度、阅读难度 |
| 人际交往标记 | `interpersonal_markers` | 礼貌策略、免责声明、模糊语以及表达立场或礼貌程度的委婉语 |

每个一级特征都必须输出描述、3 到 5 条原文例句、判断依据和置信度。项目不再把“标点风格”“话题转换”“回答详略”等作为独立一级分类；如果这些现象需要描述，只能归入上述六类中的对应类别。

第三层是结构化保存。模型返回的结果必须是严格 JSON。`analyzer.py` 会解析 JSON，并转换为 `models.py` 中定义的 `StyleProfile`。其中每个特征都是一个 `Characteristic`，包含 `key`、中文标签、特征描述、例句、证据和置信度。如果模型漏掉某个特征，解析逻辑会用“未分析”和 `0.0` 置信度兜底，保证后续流程拿到的风格画像结构稳定。

第四层是提示词生成。`prompt_builder.py` 会基于 `StyleProfile` 同时生成两版模仿提示词。版本 A 会再次调用 LLM，把风格画像整理成更自然的 system prompt，包含“说话风格总结”“回答样式要求”“参考例句”“不确定特征说明”四个部分。版本 B 不再调用模型，而是直接按固定模板拼接 `StyleProfile` 中的特征、置信度和例句，因此更可控、更便于审计。

项目对风格学习设置了一个重要边界：只学习表达格式，不学习内容立场。`analyzer.py` 和 `prompt_builder.py` 的系统提示词都明确要求不要总结话题偏好、专业方向或观点内容；例句也只用于展示句式结构、语气节奏和标点习惯。这样生成的提示词更适合用于“说话方式模仿”，而不是复刻说话人的观点或知识背景。

最后，`verifier.py` 会使用固定验证问题测试两版提示词的输出效果，例如“你今天怎么样？”、“你觉得这个项目有什么问题吗？”等。用户如果认为某些回答不像，可以通过 `feedback.py` 选择不准确的特征，并提供补充说明。系统只会重新分析被点名的特征，其他特征保持不变。这一轮反馈会更新 `StyleProfile.analysis_version`，再重新生成提示词，从而形成一个人工可控的迭代闭环。

## 4. 核心数据结构

项目核心数据结构定义在 `models.py`：

| 数据结构 | 作用 |
| --- | --- |
| `Characteristic` | 单个一级风格特征，包括 key、中文标签、描述、例句、判断依据和置信度 |
| `RawStats` | 原始统计信息，包括消息数、平均字符数和总字符数 |
| `StyleProfile` | 一份完整的说话风格画像，包含六类风格特征与版本号 |
| `MimicryPromptPair` | 两版风格模仿提示词及其对应的风格画像快照 |



## 6. 莫言风格学习结果

### 6.1 数据来源

| 项目 | 路径 |
| --- | --- |
| 输入样本 | `transcripts/inputs/input_style_moyan_yuhua_first30_moyan.txt` |
| 输出结果 | `transcripts/outputs/output_style_prompt_moyan.txt` |
| 审计文本 | `transcripts/audit/audit_moyan_yuhua_first30_moyan_timed.txt` |

该样本来自“莫言余华：和年轻人谈谈心”前 30 分钟访谈中归属于莫言的发言，已剔除主持人、余华和现场多人快速插话。

### 6.2 主要风格画像

| 分类 | 学习结果 |
| --- | --- |
| Lexical Complexity 词汇复杂度 | 平均词长约 2.0 字/词，type-token ratio 约 0.6；口语化词汇与书面语混合，双字词占主导，高级词汇不过度密集 |
| Syntactic Complexity 句法复杂度 | 平均句长约 25 字/句，以复合长句为主；从句和并列分句较多，约 2.5 个分句/句；动词、名词和助词使用较多 |
| 形式化指数 | 代词比例约 0.15，第一人称使用较多；功能词比例约 0.3；正式表达与口语表达混合，整体呈半正式口语风格 |
| Emotiveness 情感表达度 | 情感词频率约 0.1；感叹号和表情符号少见，情绪表达克制；强化词和程度副词适度出现 |
| 可读性指标 | 长句比例偏高，分句密度较高，抽象词密度中等；因复合长句和连续背景补充较多，结构清晰度中等，阅读难度中等偏高 |
| Interpersonal Markers 人际交往标记 | 常通过“当然”“我觉得”“我说”等立场标记组织表达；偶尔使用设问自答和委婉判断，礼貌策略不明显但语气整体平和 |

### 6.3 生成提示词的表达要求

莫言样本生成的提示词要求模型在模仿时：

- 使用半正式口语风格。
- 保持叙事完整性。
- 优先使用时间或地点状语作为叙述起点。
- 回答中至少包含一个明确逻辑连接词。
- 允许使用 2 到 3 个复合分句展开说明。
- 表达对立或转折时，优先使用“但是/然而 + 完整分句”的结构。
- 如使用问句，倾向采用“设问 + 自答”的结构。

### 6.4 代表性格式例句

以下例句仅用于展示说话格式，模仿时只参考句式结构和语气节奏，不应复用其中的具体内容。

> “1987年，我当时正在写一个长篇小说，《天堂蒜薹之歌》。”

> “所以我说也没有必要由我来写这种骈体文，要求对仗、要求平仄。”

> “有没有功利心？肯定是有的。”

> “当然也有比较高大上的理想和愿望。”

> “但是写作过程当中依然有很多困惑。”

### 7.2 主要风格画像

| 分类 | 学习结果 |
| --- | --- |
| Lexical Complexity 词汇复杂度 | 平均词长约 2.5 字/词，type-token ratio 约 0.6；口语化词汇较多，偶尔出现文学性表达 |
| Syntactic Complexity 句法复杂度 | 平均句长约 15 字/句，长短句混合；短句约 5 到 10 字，长句约 15 到 25 字；从句频率中等，约 1.5 个从句/句 |

| 可读性指标 | 长句比例中等，分句密度中等，抽象词密度中等；句子较莫言样本更短，段落层次更清楚，阅读难度中等 |
| Interpersonal Markers 人际交往标记 | 可用问候语开头，也常直接进入主题；常见“我觉得”“其实”“确实”等立场标记和轻度缓和语，偶尔使用反问或设问 |

### 7.3 生成提示词的表达要求

余华样本生成的提示词要求模型在模仿时：

- 使用长短句交替结构。
- 短句控制在 10 字以内，长句控制在 20 字左右。
- 每段首句尽量承担主题句功能。
- 后续内容用“所以”“但是”“因为”等连接词展开。
- 保持口语化基调，适当使用“其实”“确实”等语气词。
- 标点以逗号和句号为主，回答应自然流畅，避免过度书面化。

> “这个是我已经毕业以后拍的。”
